# AIMAG AI Analyst — виртуальный крипто-трейдер и аналитик в Telegram

Песочница: сделки виртуальные (SQLite), деньги не трогаются. ИИ предлагает идеи,
Python-цензор валидирует (плечо только 3x/5x, риск 2–5%, санитария уровней),
БД дополнительно защищена CHECK-ограничениями.

## Быстрый старт

```bash
# 1. Зависимости
pip install -r requirements.txt

# 2. Конфигурация
cp .env.example .env
# Заполнить: TELEGRAM_BOT_TOKEN (от @BotFather), TELEGRAM_CHAT_ID (свой id,
# узнать у @userinfobot), ANTHROPIC_API_KEY

# 3. Проверка сборщиков данных (по отдельности)
python -m collectors.macro     # Nasdaq, S&P500, DXY, VIX, золото
python -m collectors.crypto    # BTC/ETH/SOL + funding + свечи 4h
python -m brain.news           # RSS-ленты

# 4. Запуск бота
python main.py
```

## Команды бота

| Команда | Что делает |
|---|---|
| /report | Полный цикл: данные → индикаторы → LLM → CIO → сделки |
| /balance | Виртуальный счёт, доходность, винрейт |
| /positions | Открытые позиции с нереализованным PnL |
| /close <id> | Закрыть позицию по рынку вручную |
| /rejected | Топ причин отказа цензора |
| /reset confirm | Сброс песочницы |

Утренний брифинг приходит автоматически (по умолчанию 08:00 Asia/Almaty,
настраивается в .env). Мониторинг SL/TP — каждые 5 минут.

## Архитектура

```
collectors/  Модуль 1: yfinance (макро) + Binance Futures public API (крипта)
database/    Модуль 2: SQLite — account, trades, signals_log (чёрный ящик)
brain/       Модуль 3: индикаторы (код) + новости + LLM + CIO-скоринг + ЦЕНЗОР
bot/         Модуль 4: aiogram-хендлеры + форматирование
services/    Пайплайн-оркестратор
main.py      Бот + планировщик в одном asyncio loop
```

## Принципы безопасности

- Плечо вне (3, 5) отклоняется цензором И схемой БД (двойной эшелон).
- Маржу считает код из риск-процента; LLM цифры маржи не диктует.
- Стоп не может лежать в зоне ликвидации (запас 20%).
- Одна позиция на символ; маржа сделки ≤ 30% баланса.
- Все ответы LLM (принятые/отклонённые) пишутся в signals_log с причиной.
- Бот отвечает только владельцу (TELEGRAM_CHAT_ID), чужие запросы игнорирует.

## Известные ограничения песочницы

- SL/TP проверяются по последней цене раз в N минут — быстрый прокол уровня
  между проверками не фиксируется.
- Нет комиссий, funding-платежей и проскальзывания — реальный результат был бы хуже.
- «Композит роста» — не статистическая вероятность; честную точность покажет
  только сверка прогнозов с фактом на дистанции.
